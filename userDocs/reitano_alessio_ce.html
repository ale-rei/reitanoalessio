LAB ISS | the project cautiousExplorer

Introduction

This case-study starts to deal with the design and development of proactive/reactive software systems that use aynchronous exchange of information.

Requirements

Design and build a software system that allow the robot described in VirtualRobot2021.html to exibit the following behaviour:
  1. the robot lives in a closed environment, delimited by walls that includes one or more devices (e.g. sonar) able to detect its presence;
  2. the robot has a den for refuge, located near a wall;
  3. the robot works as an explorer of the environment. Starting from its den, the robot moves (either randomly or - preferably - in a more organized way) with the aim to find the fixed obstacles around the den. The presence of mobile obstacles is (at the moment) excluded;
  4. since the robot is 'cautious', it returns immediately to the den as soon as it finds an obstacle. Optionally, it should also return to the den when a sonar detects its presence;
  5. the robot should remember the position of the obstacles found, by creating a sort of 'mental map' of the environment.

    6. Requirement analysis

    1. robot: A device capable of moving according to what is reported in the file delivered by the customer
    2. closed environment: Empty rectangular room bounded by walls
    3. devices: Devices capable of detecting the presence of the robot
    4. den: Home of the robot or more simply its starting point, to which it returns once it finds an obstacle
    5. fixed obstacles: Object placed in the path of the robot that does not allow it to continue its path d
    6. cautious: Adjective that defines the behavior of the robot when it collides with obstacles
    Regarding actions (verbs):

    1. finds: Moment when the robot through the impact sensor shows a collision
    2. should remember: The robot remembers the position of the obstacles by creating a map of the entire room

    The main user story (validated by the customer)

    As a user, I put the robot in any starting position; subsequently, I activate the cautiousExplorer application that allows the robot to move around the room and return to its original position when it encounters an obstacle or when the sonar is activated. The application cannot be stopped by any user commands.

    Problem analysis

    We highlight that:
    1. In the VirtualRobot2021.html:commands the customer states that the robot can receive move commands in two different ways:
      • by sending messages to the port 8090 using HTTP POST
      • by sending messages to the port 8091 using a websocket
    2. In this specific case we should proceed using the websocket mode, because it is much more useful to choose an asynchronous mode
    3. With respect to the technological level, there are many libraries in many programming languages that support the required protcols.
    4. I have no specifics when it comes to the robot's movements, that is, if they have to be random or organized
